# Команды bmad-c9s (русский)

Каждый файл — краткий сценарий (c9s). Полные пошаговые тексты BMAD v6 (англ.): после `./install-v6.sh` — `~/.claude/config/bmad/bmad-v6-bundle/commands/*.md`; в репозитории — `bmad-v6/commands/*.md`.

Общие шаги перед работой:

1. Активируй соответствующий **скилл** из `bmad-c9s/skills/`.
2. Прочитай [FORMAT-CAPITAL-GITHUB.md](~/.claude/config/bmad/FORMAT-CAPITAL-GITHUB.md), [GIT-COMMITS-POLICY.md](~/.claude/config/bmad/GIT-COMMITS-POLICY.md) и [helpers-ru.md](~/.claude/config/bmad/helpers-ru.md) (или `helpers.md` в той же папке).
3. Пиши содержание в **файлы story** под **`requirements/`** (и `issues/` по канону); в **`.c9s/`** — только YAML/статусы воркфлоу, не брифы и не PRD.
4. Git: [c9s-push-results.md](c9s-push-results.md), [c9s-monorepo-component-git.md](c9s-monorepo-component-git.md).
5. Задачи и время: [tasks-documents-time.md](tasks-documents-time.md), передача на ревью мастеру: [c9s-submit-for-review.md](c9s-submit-for-review.md).
